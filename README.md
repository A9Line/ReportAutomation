# ReportAutomation — программное средство формирования отчётной документации

Настольное приложение для автоматизированного формирования и консолидации
отчётной документации на основе данных учётных информационных систем
предприятия. Разработано в рамках производственной практики на базе
АО «Сетевая компания» (г. Казань).

## Описание

На предприятии подготовка сводной отчётности связана с регулярной ручной
выгрузкой данных из учётных систем, их проверкой, сведением в единые формы
и формированием итоговых документов. Эти операции повторяются, занимают
значительное время и допускают ошибки при переносе данных.

Программное средство выполняет перечисленные операции в автоматическом
режиме. Оператор задаёт отчётный период и указывает файлы с исходными
данными, после чего приложение получает данные, проверяет их корректность,
сводит со справочником подразделений, рассчитывает сводные показатели и
формирует итоговый сводный отчёт и пояснительный документ. За оператором
остаётся контроль результата в нестандартных ситуациях.

## Возможности

- Получение исходных данных из файлов электронных таблиц формата .xlsx
- Проверка структуры входных файлов до загрузки данных
- Проверка полноты и корректности данных по набору настраиваемых правил
- Сведение данных из нескольких источников по коду подразделения
- Расчёт сводных показателей по подразделениям и итогов по предприятию
- Формирование сводного отчёта в формате электронной таблицы (Excel)
- Формирование пояснительного отчёта в формате текстового документа (Word)
- Ведение журнала операций и его экспорт в табличный вид

## Технологии

- Язык программирования: C#
- Платформа: .NET
- Графический интерфейс: Windows Presentation Foundation (WPF)
- Работа с электронными таблицами: библиотека ClosedXML
- Формирование текстовых документов: пакет Open XML SDK

## Архитектура и состав классов

Приложение построено по модульному принципу: получение данных, их
обработка и формирование документов разделены между отдельными классами,
а управляет процессом единый координирующий класс.

### Интерфейс и управление

- `MainWindow` — главное окно, графический интерфейс оператора
- `ReportController` — управление процессом и координация работы классов

### Получение и обработка данных

- `DataLoader` — получение исходных данных из файлов электронных таблиц
- `ExcelTemplateChecker` — проверка структуры входных файлов
- `DataProcessor` — проверка, сведение и расчёт показателей
- `ReportSummaryCalculator` — расчёт итоговых показателей по предприятию

### Правила проверки данных

- `IValidationRule` — интерфейс правила проверки
- `ValidationRules` — конкретные правила проверки (наличие кода, баланс и др.)
- `RuleSet` — набор правил, применяемых к данным
- `ValidationResult` — результат проверки записи

### Формирование документов и журнал

- `ReportGenerator` — формирование сводного и пояснительного документов
- `ReportTemplate` — шаблон отчётного документа
- `Logger` — ведение журнала операций
- `JournalExporter` — экспорт журнала в табличный вид

### Модели данных и настройки

- `ReportRequest` — параметры запуска (период, источники данных)
- `ReportResult` — результат формирования отчёта
- `Indicator` — производственный показатель по подразделению
- `Department` — запись справочника подразделений
- `SummaryRow` — строка сводного отчёта
- `AppConfig` — конфигурация и правила проверки данных

## Запуск

1. Открыть решение `ReportAutomation` в Visual Studio.
2. Восстановить пакеты NuGet (ClosedXML, DocumentFormat.OpenXml).
3. Собрать решение (Сборка → Собрать решение).
4. Запустить приложение.
5. В главном окне задать отчётный период, указать файл с показателями и
   файл справочника подразделений, нажать кнопку «Сформировать отчёт».

## Результат работы

Сформированные документы сохраняются в каталог «Документы\Отчёты»:
- сводный отчёт по подразделениям (Excel)
- пояснительный отчёт (Word)
- журнал операций (текстовый файл и таблица)

## Входные данные

Файл с показателями (.xlsx), столбцы: код подразделения, объём передачи,
полезный отпуск, потери. Файл справочника подразделений (.xlsx), столбцы:
код подразделения, наименование. Первая строка каждого файла — заголовки.
